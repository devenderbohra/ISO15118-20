About the ISO15118-20 Protocol

ISO15118-20 represents the latest evolution in electric vehicle charging communication standards, introducing revolutionary capabilities that transform the EV charging experience through enhanced security, bidirectional power transfer, and seamless user experiences.

Protocol Overview

The ISO15118-20 standard defines the communication interface and protocol between electric vehicles (EVs) and EV supply equipment (EVSE) for both conductive and wireless power transfer. This standard enables sophisticated energy management, secure authentication, and automated billing processes.

Key Objectives

  • Enable seamless Plug & Charge functionality with enhanced security
  • Support bidirectional power transfer for Vehicle-to-Grid (V2G) applications
  • Provide robust security through mutual authentication and encryption
  • Support wireless power transfer communication protocols
  • Ensure global interoperability across different charging networks

Key Improvements Over ISO15118-2

Enhanced Security

Upgraded from secp256r1 to secp521r1 or Ed448 cryptographic curves, with mutual TLS 1.3 authentication providing quantum-resistant security foundations.

Bidirectional Power Transfer

Native support for Vehicle-to-Grid (V2G), Vehicle-to-Home (V2H), and Vehicle-to-Load (V2L) applications through structured power flow management.

Multiple Contracts

Support for multiple charging contracts and service providers within a single charging session, enabling complex energy service arrangements.

Wireless Integration

Comprehensive support for wireless power transfer (WPT) communication, enabling automated parking and charging systems.

Core Protocol Components

1

Physical Layer (ISO15118-3)

Defines the physical connection interface including PLC (Power Line Communication) and wireless communication methods.

2

Network & Transport Layer (ISO15118-4)

Manages network communication protocols including IPv6, TCP, and UDP for reliable data transmission.

3

Application Layer (ISO15118-20)

Defines the high-level communication protocol including message formats, authentication, and charging control.

4

Security Layer (ISO15118-5)

Comprehensive security framework including PKI infrastructure, certificate management, and encryption protocols.

Communication Flow

The ISO15118-20 communication process follows a structured sequence of interactions between the EV and charging station:

Charging Session Sequence

  1. Connection & Detection: Physical connection established and communication initiated
  2. Authentication & Authorization: Certificate-based mutual authentication between EV and EVSE
  3. Service Discovery: Available charging services and capabilities negotiated
  4. Contract Certificate Installation: Digital contracts and payment certificates provisioned
  5. Charging Parameter Exchange: Power requirements and constraints communicated
  6. Power Delivery: Controlled energy transfer with continuous monitoring
  7. Session Termination: Secure session closure with billing information exchange

Technical Specifications

Communication Protocol

  • Transport: TCP/UDP over IPv6
  • Encoding: EXI (Efficient XML Interchange)
  • Security: Mutual TLS 1.3
  • Certificates: X.509 v3 with enhanced key sizes

Power Transfer

  • AC Charging: Single and three-phase
  • DC Charging: Up to 1000V, 1000A
  • Bidirectional: Full V2G capability
  • Wireless: Inductive and resonant coupling

Security Features

  • Cryptographic Curves: secp521r1, Ed448
  • Hash Algorithms: SHA-256, SHA-3
  • Certificate Validation: OCSP, CRL
  • Key Management: Automated provisioning

Service Capabilities

  • Multiple Contract Support
  • Dynamic Pricing Integration
  • Load Management
  • Grid Services Integration

Implementation Considerations

Backward Compatibility

While ISO15118-20 introduces significant improvements, it maintains compatibility pathways with ISO15118-2 systems. However, full feature compatibility requires infrastructure upgrades due to enhanced security requirements and new message formats.

Certification Requirements

Implementation of ISO15118-20 requires comprehensive certification covering protocol conformance, interoperability testing, and security validation. This ensures reliable operation across different vendor implementations.

Infrastructure Requirements

Successful deployment requires updated PKI infrastructure, enhanced hardware security modules (HSMs), and network infrastructure capable of supporting the increased security and communication requirements.
